# 서울 버스 관제 용어 사전

| **한국어** | **영어** | **설명** |
| --- | --- | --- |
| 관제 | Monitoring | 운행 중인 버스의 위치, 통신 상태, 이벤트, 영상 수신 상태를 한 화면에서 확인하는 업무 |
| 관제 담당자 | Operator | 관제 화면을 통해 버스 운행 상태와 이벤트를 확인하는 사용자 |
| 버스 | Bus | 관제 대상이 되는 운행 차량. 하나의 노선 방향 패턴 위에서 운행한다 |
| 버스번호 | Bus Number | 관제 시스템 안에서 버스를 구분하기 위한 번호. 예: `7016-01` |
| 차량번호 | Plate Number | 실제 차량 등록번호. 예: `서울70사1234` |
| 노선 | Route | 노선번호와 노선명으로 식별되는 버스 운행 노선 자체 |
| 노선번호 | Route Number | 승객과 관제자가 노선을 식별하는 공개 업무 식별자. 예: `7016`, `143`, `N26`, `M7106` |
| 노선명 | Route Name | 사람이 읽을 수 있는 노선 설명. 예: `은평공영차고지 ↔ 상명대` |
| 노선 방향 | Route Direction | 하나의 노선 안에서 기점/종점 방향으로 구분되는 운행 패턴 |
| 노선 방향 구분 | Route Direction Type | 노선 방향을 구분하는 값. 예: `OUTBOUND`, `INBOUND` |
| 기점 | Origin | 특정 노선 방향 패턴이 시작되는 지점 |
| 종점 | Destination | 특정 노선 방향 패턴이 끝나는 지점 |
| 노선 정류장 | Route Stop | 특정 노선 방향에 포함된 정류장과 그 순서 |
| 정류장 | Stop | 버스가 승하차를 위해 정차하는 위치 |
| 정류장번호 | Stop Number | 외부에서 정류장을 식별하는 번호 |
| 정류장 순서 | Stop Order | 특정 노선 방향 안에서 정류장이 등장하는 순서 |
| 현재 정류장 | Current Stop | 버스가 현재 위치한 것으로 판단되는 정류장 |
| 다음 정류장 | Next Stop | 현재 정류장 이후 버스가 향하는 다음 정류장 |
| 버스 위치 | Bus Position | 특정 시각에 기록된 버스의 GPS 위치와 속도 정보 |
| 최신 위치 | Latest Position | 한 버스의 위치 이력 중 기록 시각이 가장 최신인 위치 |
| 위치 좌표 | Coordinate | 위도와 경도로 표현되는 지도상 위치 |
| 현재속도 | Current Speed | 최신 위치에 기록된 버스 속도 |
| 마지막 통신시간 | Last Communication Time | 버스 단말 또는 차량 시스템이 관제 서버와 마지막으로 통신한 시각 |
| 버스 통신 상태 | Bus Communication Status | 마지막 통신시간 기준으로 계산되는 버스의 온라인/오프라인 상태 |
| 온라인 | Online | 마지막 통신시간이 현재 시각 기준 5분 이하인 상태 |
| 오프라인 | Offline | 마지막 통신시간이 현재 시각 기준 5분 초과인 상태 |
| 버스 이벤트 | Bus Event | 급정거, 급가속, 충격 등 관제 담당자가 확인해야 하는 운행 이벤트 |
| 이벤트 유형 | Event Type | 이벤트의 종류. 예: 급정거, 급가속, 급감속, 충격 |
| 이벤트 심각도 | Event Severity | 이벤트의 중요도 또는 위험 수준. 예: 낮음, 보통, 높음 |
| 급정거 | Sudden Stop | 주행 중 급격히 감속하거나 정차한 이벤트 |
| 급가속 | Sudden Acceleration | 짧은 시간에 속도가 급격히 증가한 이벤트 |
| 급감속 | Sudden Deceleration | 주행 중 속도가 급격히 감소한 이벤트 |
| 충격 | Impact | 차량에 물리적 충격이 감지된 이벤트 |
| 버스 카메라 상태 | Bus Camera Status | 버스별 카메라의 영상 수신 여부와 마지막 수신시간 |
| 카메라 유형 | Camera Type | 버스에 설치된 카메라 구분. 예: 전면, 후면, 내부 CCTV |
| 전면 블랙박스 | Front Camera | 차량 전방을 촬영하는 카메라 |
| 후면 블랙박스 | Rear Camera | 차량 후방을 촬영하는 카메라 |
| 내부 CCTV | Interior Camera | 버스 내부를 촬영하는 카메라 |
| 영상 수신 여부 | Receiving | 특정 카메라의 영상이 정상적으로 수신되고 있는지 여부 |
| 스트림 URL | Stream URL | 영상 스트림을 조회하기 위한 주소. MVP에서는 mock 값을 사용한다 |
| 최근 이벤트 목록 | Recent Events | 발생시간 기준 최신순으로 조회한 이벤트 목록 |
| 더미 데이터 | Mock Data | 실제 데이터가 제공되지 않는 과제 조건에서 직접 생성한 테스트/시연용 데이터 |
| 폴링 | Polling | 프론트엔드가 일정 주기로 API를 다시 호출해 최신 상태를 갱신하는 방식 |
